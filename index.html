Game Engine

After reading Game Engine Architecture, I felt compelled to apply what I learned and make my own 3D game engine. Written entirely in C++, I followed many industry best practices to create an engine that is fast and efficient. In addition, I developed a small game to make sure I had created all the necessary features and that they all worked together.

Highlights include:

  • An efficient and custom Entity Component System [Entity.h] | [ComponentSystem.h]
  • A pool structure allowing for fast iteration of objects through contiguous memory [CompactPool.h]
  • Rendering through direct use of the DirectX 11 API [Graphics.cpp]
  • Custom HLSL shader implementing Phong Shading [tutorial6.hlsl]
  • Physics using the Bullet Physics Engine API [Physics.cpp]
  • A small platformer combining all the features of the engine [ThirdPersonApp.h]

Other Projects

Crash Report

While working at Ready At Dawn, I wrote a Python script that helped programmers identify which crashes affected the largest portion of the Echo VR player base within a specific time frame. This helped the team prioritize the crashes that needed to be fixed first. The script pulled crash data from Bug Splat and player data from Kibana. It used the data to calculate the percentage of players affected by each crash and then generated an email report, complete with graphs showing the crash trends over time.

QA Summary Report

I wrote a Python script for the Ready At Dawn QA team that generated emails summmarizing the team’s daily activities. It pulled data from Jira, and it organized and presented the data in attractive tables. Also, since each project had different needs, I allowed the script to be customized, both in presentation and content, by modifying a JSON file.
